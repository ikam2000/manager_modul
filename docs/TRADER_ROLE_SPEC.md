# Спецификация: роль «Трейдер» и модуль торговли

## Контекст бизнес-модели

**Zero-stock модель:** закупка товаров у поставщителей близких к сроку годности со скидкой, продажа со скидкой пользователям, минимум собственных складов.

**Текущая боль:**
- Получение актуального стока от поставщиков — запросы вручную, данные приходят в разных форматах (документы, Excel)
- Расчёт маржи по price engineering (размер скидки → покрытие логистики, маркетинга)
- Обновление в Shopify по баркодам: цены, остатки, сроки годности
- Всё вручную → медленно, ошибки, нет единого актуального источника данных
- Нет дашборда по продажам, затратам, прибыли, убыткам и их причинам

## 1. Роль «Трейдер»

### 1.1 Сущности и права

- **Новая роль:** `trader` — отдельная от `user`, `admin`, `super_admin`
- Трейдер работает со **своей номенклатурой** (изолированной от обычной номенклатуры компании или в отдельном контексте)

### 1.2 Номенклатура трейдера

- **Свойства:** код, баркод, название, категория, подкатегория, поставщик, срок годности, закупочная цена, наценка, итоговая цена, остаток
- **Разбивка:** по поставщикам, по категориям товаров
- **Управление наценкой:**
  - по поставщику (глобальная наценка для поставщика)
  - по категории товаров
  - по отдельному товару (переопределение)

### 1.3 Источники данных от поставщиков

| Источник        | Формат                  | Режим                          |
|-----------------|--------------------------|--------------------------------|
| API             | JSON/REST                | По расписанию (CRON)           |
| OAuth           | API провайдера           | По расписанию (CRON)           |
| Excel-файлы     | Свой столбцы у поставщика| Ручная загрузка / импорт       |

- **Нормализация:** маппинг полей поставщика → единый справочник (баркод, название, цена, срок годности, остаток)
- **Режимы импорта:** добавление новых товаров; обновление существующих по баркоду/коду

### 1.4 Выгрузка на торговые площадки

- **Целевые платформы:** Shopify (уже есть OAuth), Wildberries, Ozon — по мере настройки
- **Применение наценки** перед выгрузкой
- **Формат выгрузки:** по API площадки (цены, остатки, срок годности где поддерживается)
- **Расписание:** отдельный CRON на выгрузку (независимо от получения от поставщиков)

### 1.5 CRON / расписания

- **Получение от поставщиков:** настраиваемое расписание (каждые N часов/дней) для API/OAuth-поставщиков
- **Выгрузка на площадки:** отдельное расписание (частота обновления цен и остатков)

## 2. Дашборды аналитики для трейдера

### 2.1 KPI

- **Оборачиваемость** — средний срок от закупки до продажи
- **Общие продажи** — сумма выручки по периоду
- **Прибыль** — выручка минус закупочная стоимость
- **Маржинальность** — прибыль / выручка
- **Убытки** — товары с истекшим сроком годности, списанные
- **Поставщики** — прибыль/убыток по поставщику
- **Категории** — прибыль/убыток по категории

### 2.2 Визуализация

- Графики по периодам (день/неделя/месяц)
- Таблицы: топ поставщиков, топ категорий, проблемные товары (низкая оборачиваемость, близкий срок годности)

---

## 3. Выгрузка в Excel

- Экспорт товаров с расчётом наценки
- Колонки: баркод, название, категория, поставщик, закупочная цена, наценка (%), итоговая цена, остаток, срок годности
- Возможность фильтра по поставщику, категории, наличию остатка

## Этапы реализации

| Этап | Описание |
|------|----------|
| 1 | Роль `trader`, изоляция данных (trader_company или флаг у компании) |
| 2 | Номенклатура трейдера: модель, CRUD, наценки по поставщику/категории/товару |
| 3 | Импорт Excel с маппингом полей (добавление + обновление по баркоду) |
| 4 | Расширение OAuth/API: получение данных от поставщиков (универсальные адаптеры) |
| 5 | CRON: получение от поставщиков, выгрузка на площадки |
| 6 | Выгрузка на Shopify/WB/Ozon с применением наценки |
| 7 | Дашборд трейдера: KPI, графики, таблицы |
| 8 | Экспорт в Excel с наценкой |

## Связь с текущей схемой ikamdocs

- **Company.company_type:** уже есть `supplier`, `customer` → добавить `trader`
- **Role:** добавить `trader` в enum (или использовать `admin` в компании типа trader)
- **OAuth:** уже есть Shopify, Wildberries, Ozon — использовать для выгрузки
- **Номенклатура:** общая модель `Nomenclature` + связь с `company_id` — трейдер видит только свою
