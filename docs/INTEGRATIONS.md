# Интеграции ikamdocs

Обзор способов интеграции с внешними системами: REST API, Webhooks, модули 1С/ERP/CRM, автоматизация (Zapier, Make, n8n).

---

## 1. REST API

### Назначение

Двухсторонняя передача данных с 1С, ERP, CRM и другими системами.

### Получение API-ключа

1. Личный кабинет → **Интеграции** → вкладка **REST API**
2. Создайте новый ключ (например, «1С УТ 11»)
3. Сохраните ключ — он отображается **один раз**

### Базовый URL

```
https://ikamdocs.ru/api/v1/integrate
```

(На тестовом окружении используйте ваш домен.)

### Аутентификация

Все запросы должны содержать заголовок:

```
X-Api-Key: ваш_api_ключ
```

### Сущности

| entity_type       | Описание                    |
|-------------------|-----------------------------|
| category          | Категория                   |
| subcategory       | Подкатегория                |
| nomenclature      | Номенклатура                |
| supplier          | Поставщик                   |
| manufacturer      | Производитель               |
| supply            | Поставка                    |
| contract          | Договор поставки            |
| contract_appendix | Приложение к договору       |

### Эндпоинты

| Метод  | Путь                              | Описание              |
|--------|-----------------------------------|-----------------------|
| GET    | /entities?entity_type=…&limit=&offset= | Список сущностей |
| POST   | /entities                         | Создание сущности     |
| GET    | /entities/{entity_type}/{id}      | Получение по ID       |
| PUT    | /entities/{entity_type}/{id}      | Обновление            |
| DELETE | /entities/{entity_type}/{id}      | Soft delete           |

### Пример POST (создание номенклатуры)

```json
{
  "entity_type": "nomenclature",
  "data": {
    "name": "Изделие А",
    "code": "IZD-001",
    "category_id": 1,
    "tag_number": "TAG-123",
    "specification": "..."
  }
}
```

## 2. Webhooks

### Назначение

Push-уведомления при изменениях сущностей. ikamdocs отправляет POST на ваш endpoint.

### Настройка

1. Личный кабинет → **Интеграции** → вкладка **Webhooks**
2. Добавьте URL (https://…)
3. Выберите события: номенклатура создана/обновлена/удалена, поставка, поставщик, договор

### События

| Событие                | Описание                    |
|------------------------|-----------------------------|
| nomenclature.created   | Номенклатура создана        |
| nomenclature.updated   | Номенклатура обновлена      |
| nomenclature.deleted   | Номенклатура удалена        |
| supply.created         | Поставка создана            |
| supply.updated         | Поставка обновлена          |
| supplier.created       | Поставщик создан            |
| supplier.updated       | Поставщик обновлён          |
| contract.created       | Договор создан              |
| contract.updated       | Договор обновлён            |

### Формат payload (POST)

```json
{
  "event": "nomenclature.created",
  "entity_type": "nomenclature",
  "entity_id": 123,
  "timestamp": "2026-02-25T12:00:00Z",
  "data": { ... }
}
```

## 3. Модули 1С / ERP / CRM

### 1С:Предприятие

- Обработка для синхронизации справочников и документов
- HTTP-соединение к REST API
- Маппинг полей согласно конфигурации 1С
- **Доступ:** установочный файл по запросу или магазин конфигураций 1С

### ERP / CRM (AmoCRM, Bitrix24 и др.)

- Интеграция через REST API и webhooks
- Zapier, Make (Integromat), n8n — готовые сценарии
- API-ключ и webhooks для двусторонней синхронизации

### Zapier / Make / n8n

- Автоматизация без кода
- Триггеры на события (webhooks)
- Действия: создание, обновление сущностей через REST API

---

## 4. Рекомендации

1. **Безопасность:** храните API-ключ в переменных окружения, не в коде
2. **Идемпотентность:** при повторных вызовах используйте код изделия для дедупликации
3. **Ограничения:** соблюдайте лимиты (limit до 200 на запрос)
4. **Версионирование:** API v1 стабилен; возможные изменения будут в новых версиях
