# 7. Описание состава ПО

Документ описывает состав поставки **ikamdocs** на уровне компонентов и зависимостей. Модули, относящиеся исключительно к интеграции с внешними торговыми площадками, в эксплуатационное описание **не** включаются; при наличии в репозитории они могут поставляться «как есть» без обязательств по сопровождению в рамках настоящего комплекта.

---

## 7.1. Логическая архитектура

```
[Браузер] → Nginx (TLS, статика SPA)
              ├──→ GET / → frontend/dist (React SPA)
              └──→ proxy → Uvicorn (FastAPI) :8000
                        ├── PostgreSQL
                        ├── Redis (опционально, кэш/сессии)
                        └── ФС: STORAGE_PATH (документы)
```

---

## 7.2. Состав компонентов продукта

| Компонент | Технология / расположение | Назначение |
|-----------|---------------------------|------------|
| **Frontend SPA** | React 18, TypeScript, Vite 5; каталог `frontend/` | Маркетинговые страницы, кабинет, маршрутизация |
| **Backend API** | FastAPI, Uvicorn; `backend/app/` | REST API, авторизация, бизнес-логика |
| **Слой данных** | SQLAlchemy 2 async, asyncpg | Доступ к PostgreSQL |
| **Миграции БД** | SQL-файлы `backend/migrations/`, скрипт `migrate.sh` | Эволюция схемы |
| **Фоновые задачи** | APScheduler, при необходимости отдельный worker | Периодические задания |
| **Reverse proxy** | Nginx; пример `deploy/nginx-ikamdocs.conf` | HTTPS, раздача `dist`, прокси к API |
| **Процессы** | systemd (примеры в `deploy/*.service`) | Автозапуск backend/worker |

---

## 7.3. Основные модули backend (пакет `app`)

| Модуль / роутер | Назначение |
|-----------------|------------|
| `main.py` | Точка входа FastAPI, middleware, подключение роутеров |
| `routers/auth` | Регистрация, вход, JWT, профиль |
| `routers/cabinet` | Личный кабинет: компания, ключи, webhooks и др. |
| `routers/entities` | Номенклатура, категории, поставщики, договоры и связанные сущности |
| `routers/documents` | Загрузка и обработка документов |
| `routers/api_integration` | REST API для внешних систем (`X-Api-Key`) |
| `routers/qr` | QR-маркировка |
| `routers/analytics` | Аналитика |
| `routers/payment` | Платежи и webhook ЮKassa |
| `routers/admin` | Администрирование платформы |
| `routers/public` | Публичные эндпоинты |
| `routers/trader` | Контур торгового представителя |
| `database`, `models`, `security` | БД, ORM-модели, хеширование паролей, JWT |

В репозитории могут присутствовать маршруты внешних OAuth-провайдеров; в рамках настоящей поставки без маркетплейсов они не описываются как обязательная функциональность.

---

## 7.4. Зависимости runtime

### Python (`backend/requirements.txt`)

Основные группы: FastAPI, Uvicorn, SQLAlchemy, asyncpg, Alembic (зависимость), python-jose, bcrypt, cryptography, redis, httpx, yookassa, aiosmtplib, prometheus_client, slowapi, apscheduler, openpyxl, python-docx, pypdf, reportlab и др. Полный перечень — в файле.

### Node (`frontend/package.json`)

Основные: React, react-router-dom, Vite, TypeScript, TanStack Query, Zustand, Recharts, lucide-react и др. Полный перечень — в файле.

### Системное ПО

PostgreSQL, Nginx, Redis (по необходимости), Certbot для TLS.

## 7.6. Лицензии сторонних библиотек

Зависимости подбираются из экосистемы open-source (PyPI, npm). Точные лицензии пакетов можно получить командами:

```bash
cd backend && pip install pip-licenses  # при необходимости
cd frontend && npm ls --all
```

Для юридического аудита рекомендуется сформировать отчёт на момент релиза.

---

## 7.7. Исключения из описания состава (по соглашению сторон)

- Автоматическая синхронизация каталога с внешними торговыми площадками.
- Специализированная поддержка конкретного маркетплейса как часть гарантийной поддержки.

При необходимости перечень исключений фиксируется в договоре на передачу прав.
