# IKAMDOCS Cabinet — безопасная доработка в Cursor (без поломок)

Цель:
- Поднять визуальный уровень кабинета до enterprise (дороже, чище, единообразно)
- НЕ сломать существующие страницы/классы (`btn-*`, `analytics-*`, `data-list`, etc.)
- Минимизировать inline styles в TSX и убрать стили-хаки `style*="..."`

## Принципы (чтобы ничего не поломалось)
1) **Не трогаем** существующие классы и базовые компоненты в первом проходе.
2) Вводим **новый слой** дизайн-системы `ds-*` (scoped классы) и используем их **только на новых/рефакторенных страницах**.
3) Для старых компонентов/страниц — максимум добавляем **алиасы** (при необходимости).
4) Изменения делаем маленькими коммитами: CSS -> страницы design -> затем постепенно остальные страницы.

## Что в этом пакете
- `index_css_patch_v2.txt` — вставить в конец `index.css` (совместимо с вашим набором переменных).
- Пять TSX страниц дизайн-секции:
  - `CabinetDesignIndex.tsx`
  - `CabinetDesignTypography.tsx`
  - `CabinetDesignComponents.tsx`
  - `CabinetDesignLayout.tsx`
  - `CabinetDesignAnalytics.tsx`
- `REFACTOR_PLAN.md` — пошаговый план, что делать дальше.
- `SAFETY_CHECKLIST.md` — чек-лист, как проверять, что ничего не сломалось.
- `CURSOR_PROMPT.txt` — готовый промт для Cursor, чтобы он работал аккуратно.

## Вставка CSS патча (без конфликтов)
1) Открой `src/index.css` (или где у вас переменные и базовые стили).
2) В самый конец файла вставь содержимое `index_css_patch_v2.txt`.
3) Убедись, что переменные есть (у вас они уже есть):
   `--bg --surface --surface-hover --border --border-light --accent --accent-soft --accent-muted --text --text-secondary --text-muted --success --radius --radius-lg --radius-xl --shadow --shadow-light`

## Замена TSX файлов (только дизайн-секция)
Заменить содержимое ваших файлов на версии из папки `tsx/`.
Важно: это касается **только** `/cabinet/design/*` страниц, чтобы не затронуть прод-страницы кабинета.

## Рекомендованный порядок работ в Cursor
Шаг 1. Вставить CSS патч v2, прогнать UI
- Открыть дизайн-секцию, проверить карточки/таблицы/кнопки
- Если где-то слишком светло/темно — правим только токены/пара мелких правил, не переписываем всё

Шаг 2. Убрать опасные селекторы в кабинете (в отдельном коммите)
Сейчас у вас есть селекторы вида:
- `.cabinet-area button[style*="var(--accent)"]...`
Заменить их на классы:
- `.cabinet-area .btn-primary ...`
- `.cabinet-area .btn-secondary ...`
Если есть кастомные кнопки — добавить им класс, а не rely on inline style.

Шаг 3. Постепенно применять `ds-*` на остальных страницах
- Сначала: dashboards / analytics
- Потом: таблицы / формы / модалки

## Быстрая проверка (до пуша)
- Night/day темы переключаются
- Hover/active состояния не “прыгают”
- Таблицы не теряют sticky header (где нужно)
- Ничего не “растягивает” grid на мобиле (<= 480px)
